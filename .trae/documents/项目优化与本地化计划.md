# 项目优化与本地化计划 (最终版)

确认图片资源已存在于本地 (`src/assets/images/`)，我将基于此进行开发。

## 1. 图片资源集成
- **动态加载策略**：不修改 JSON 数据，而是通过 `src/utils/imageUtils.ts` (新建) 封装图片路径解析逻辑，根据干员名称动态加载：
  - 头像：`src/assets/images/[Name]_avatar.png`
  - 立绘：`src/assets/images/[Name]_full.png`
- **应用场景**：
  - **Banner**: 展示当期 UP 干员的**立绘**。
  - **抽卡结果**: 展示所有获得干员的**头像**。
  - **干员名册**: 展示已获得干员的**头像**。

## 2. 界面汉化与 UI 优化
- **全局汉化**：将 `App.tsx` 和 `ResultModal.tsx` 中的所有英文文案（如 "MONITOR", "PITY", "SEARCH"）替换为对应的中文术语。
- **进度显示修复**：修改 `App.tsx` 中的奖励进度显示，使用 `Math.min(current, 60)` 确保数值不超过上限。

## 3. 结果展示优化 (`ResultModal`)
- **视觉升级**：使用头像图片替换文字占位。
- **限定特效**：为 `isLimited: true` 的干员添加特殊的 CSS 动画（金色流光边框 + 粒子光效），增强获得感。

## 4. 历史记录与名册 (`HistoryArchive`)
- **新建组件** `src/components/HistoryArchive.tsx`，包含两个标签页：
  - **寻访记录**：表格形式展示所有历史抽卡记录（时间、干员、星级、卡池）。
  - **干员名册**：网格形式展示干员头像，按稀有度排序，角标显示潜能（重复获取次数），未获得干员显示置灰状态。

## 5. 执行计划
1. 创建 `imageUtils.ts`。

4. 修改 `ResultModal.tsx` (汉化、图片展示、限定特效)。
5. 验证效果。
